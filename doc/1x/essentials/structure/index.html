Arquivos da raíz do projeto 

            
public
src
nimble.json
package.json
tsconfig.json

Os arquivos nimble.json, package.json e tsconfig.json são essenciais para o projeto, leia abaixo:

• nimble.json arquivo que define se é um projeto Nimble, por isso é um arquivo obrigatório. Nele, contem diversas definições e configurações de funcionamento do projeto. Uma das grandes funcionalidades deste arquivo, é inserir o caminho de arquivos JS/TS ou CSS/SCSS para serem inseridos no projeto automaticamente, como por exemplo: "node_modules/bootstrap/dist/css/bootstrap.css".

• package.json é aonde ficam centralizadas as dependências que o seu projeto utiliza. Cada uma das dependências possuem sua versão, e elas podem conter conteúdos desde códigos em JavaScript/TypeScript até Styles (css ou scss). Por exemplo, a dependência principal do Nimble, chamada "@nimble-ts/core", é uma delas que está presente neste arquivo, já que ela é necessária para o funcionamento do seu projeto.

• tsconfig.json quando este arquivo está presente, significa que o projeto utiliza TypeScript (que é o caso do Nimble), e o conteúdo deste arquivo especifica os arquivos raiz e as opções e configurações necessárias para compilar o projeto.

Diretório public 

            
public
├─ assets
│  ├─ css
│  ├─ fonts
│  ├─ img
│  └─ js
├─ favicon.ico
└─ index.html

O diretório public é onde ficará tudo o que se refere a sua página quando estiver publicada ou após estar "rodando" em sua máquina durante seu desenvolvimento. Isso inclui o index.html e quaisquer outros arquivos físicos que sua página precise ter, como: icones, imagens, fonts, arquivos css ou js de terceiros.

É importante ressaltar que se houver arquivos css e js contidos neste diretório, eles só poderão ser utilizados/referenciados no index.html, pois para outros vindos de dependências, como node_modules, devem ser utilizados/referenciados em src/configuration.json que será abordado mais abaixo.

⚠️ Importante: o arquivo index.html é o único que é obrigatório neste diretório.

Diretório src 

            
src
├─ app
├─ environments
├─ scss
├─ main.ts
└─ style.scss

O diretório src é o "coração" de todo o desenvolvimento do projeto. É nele que fica toda a implementação, do HTML ao código TypeScript. Como visto na estrutura de diretórios acima, dentro deste existem alguns outros diretório, que serão abordados e tópicos a seguir, mas agora vamos abordar os arquivos que estão na raíz deste diretório e saber para que servem:

• index.ts arquivo em TypeScript responsável por não só inicializar o projeto em Nimble, mas também informar diversas dependencias do projeto, tais como: rotas, providers, diretivas e outros.

• style.scss onde é centralizado todos os arquivos scss (exceto das pages, que será abordado em seguida) contidos dentro do diretório src/scss, que será abordado nos próximos tópicos.

Este diretório concentra todas as páginas, providers, diretivas e também o arquivo de rotas, veja de detalhadamento sobre os dois abaixo:

• pages diretório onde as páginas devem ser centralizadas, como visto na estrutura apresentada acima,.

• routes.ts arquivo em TypeScript, obrigatório, para informar as rotas das páginas do seu site/aplicação.

Diretório src/environments 

            
src
├─ app
├─ environments
│  ├─ env.dev.js
│  ├─ env.local.js
│  └─ env.prod.js
├─ scss
├─ main.ts
└─ style.scss

Arquivos que podem conter informações para serem embarcados no projeto ao executa-lo ou em um processo de build. Estes arquivos devem sempre manter o seguinte padrão: env.NAME.ts, onde NAME é o nome específicado da sua environment. Inicialmente, após criar um projeto Nimble, vêm com estes três arquivos citados acima, para usar no projeto.

Diretório source (src/scss) 

            
src
├─ app
├─ environments
├─ scss
│  ├─ reset.scss
│  └─ variables.scss
├─ main.ts
└─ style.scss

O diretório src/scss é um diretório criado pelo Nimble para centralizar os arquivos de estilo separadamente para serem importados no style.scss (como visto anteriormente) ou em páginas caso possuam recursos a serem compartilhados. Neste projeto exemplo, existem dois arquivos sass:

• reset.scss geralmente usado para definir estilos padrões para tags HTML como: body, p, h1 e etc.

• variables.scss variáveis que são interpretedas pelo sass, e que podem ser usadas desde qune sejam importadas. Nele, por exemplo, é possível criar uma variável para guardar uma cor, e ser replicada em todos as páginas do seu site.
