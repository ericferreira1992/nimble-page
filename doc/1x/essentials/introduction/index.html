Introdução

O que é o Nimble?

Nimble é um framework de UI em TypeScript voltado para o desenvolvimento de websites, mas que também consegue criar aplicações robustas com bastante facilidade. Ele não é muito diferente dos demais frameworks ou libs de UI existentes, mas se torna uma ferramenta fácil ao trabalhar "orientado à páginas". É importante lembrar que apesar do Nimble resolver o problema do SEO, ele trabalha como uma SPA (Single Page Application).

Nimble pode ser considerado um framework onde o desenvolvimento é orientado a rotas e páginas, pois conforme a imagem abaixo é possível ver um exemplo de como é possível "montar" uma estrutura de rotas com suas respectivas páginas. Outro ponto a observar é a primeira página com o caminho da rota definida como vazia (path: ""), tendo as demais páginas como "filhas" dela, isso significa que todas herdarão o conteudo da página "pai".

Neste exemplo, o site terá as seguintes rotas: "home", "about", "contact", "admin/login", "admin/dashbord".

Primeiros passos(Instalar)

Como o Nimble é um framework, você precisará instalar sua CLI e posteriormente criar um projeto. Portanto, os exemplos a seguir considerará um projeto já criado. Caso ainda não tenha instalado a Nimble CLI, veja como clicandoaqui.

Renderizando uma página simples

01- Primeiramente, criaremos uma página com ajuda da CLI. Execute no terminal nb, aguarde as opções aparecerem e em seguida selecione Generate, depois Page e por fim digite o caminho da página. Feito isso, terá algo parecido com o exemplo abaixo:

$ nb
> Select what do you want:Generate
> Which what do you want to generate?Page
> What is path and name? (ex.: pages/home)pages/my-first
CREATED:src/app/pages/my-first/my-first.page.html
CREATED:src/app/pages/my-first/my-first.page.scss
CREATED:src/app/pages/my-first/my-first.page.ts

02- Definiremos uma rota para a página no arquivo de rotas localizado em src/app/routes.ts:

import { RouteBase } from '@nimble-ts/core/route';

export const ROUTES: RouteBase[] = [
    {
        path: 'my-first',
        page: () => import('./pages/my-first/my-first.page').then(x => x.MyFirstPage)
    }
]

03- Definindo o template (HTML):

<div class="my-first-page">
    My first <span class="purple">Nimble</span> page :)
</div>

04- Definindo o estilo (CSS):

.my-first-page {
    .purple {
        color: purple;
    }
}

05- No código (Script) da página, neste caso, não será preciso mexer, deixaremos como foi criado:

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({
    template: require('./my-first.page.html'),
    style: require('./my-first.page.scss')
})
export class MyFirstPage extends Page {

    constructor() {
        super();
    }

}

05.1 -Para ver como ficou, executaremos o nb serve no terminal para incializar um servidor de desenvolvimento:

$ nb serve

❯ Environment from:src/environments/env.local

❯ Await, starting the server...

05.2 -Abrindo o navegador no endereço http://localhost:8090/my-first, teremos o resultado abaixo:

fiber_manual_record
http://localhost:8090/my-first
Minha primeira página em Nimble :)

Renderização declarativa

O Nimble permite renderizar dados no DOM (Document Object Model) usando sintaxes muito fáceis, possibilitando customizar de forma dinâmica as páginas desde textos até priedades dos elementos, veja os exemplos abaixo

01- Definindo uma propriedade nova na classe da página:

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({ ... })
export class MyFirstPage extends Page {

    public myDynamicText: string = 'Eric';

    constructor() {
        super();
    }

}

02- Preparando o template para renderizar nosso texto dinâmico:


<div class="my-first-page">
    Hello {{myDynamicText}} :)
</div>

03- Resultado:

fiber_manual_record
http://localhost:8090/my-first
Hello Eric :)

Atualizando a página com novas informações

Para que as alterações de código de uma página sejam refletidas diretamente na página (após ela ter sido renderizada), é necessário dizer ao framework de que ela deve ser atualizada, simplesmente invocando o método this.render().

⚠️ Importante: o Nimble utiliza um algoritmo bem eficiente para saber exatamente o que foi modificado e atualizar de forma pontual, tornando isto o mais performático possível.

01-Definindo na classe da página uma propriedade chamada seconds para ser o temporizador, onde esta terá seu valor incrementado com +1 a cada segundo e que em seguida solicitará que a página seja atualizada para mostrar o valor:

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({ ... })
export class MyFirstPage extends Page {

    private timer: any;
    public seconds: number = 0;

    constructor() {
        super();
    }

    onInit() {
        this.timer = setInterval(() => {
            this.seconds += 1;
            this.render();
        }, 1000);
    }

    onDestroy() {
        clearInterval(this.timer);
    }

}

02- Colocando no template a propriedade deverá ser renderizada:


<div class="my-first-page">
    Já fazem {{seconds}} segundos que esta página foi renderizada.
</div>

03- Resultado:

fiber_manual_record
http://localhost:8090/my-first
Já fazem 1 segundos que esta página foi renderizada.

Condições e laços de iteração

Fácilmente é possível determinar se tal elemento será incluído ou não na sua página, além de poder realizar estruturas de repetições. Para isto usamos diretivas de iterações nativas presentes no framework, veja os exemplos a seguir.

Exibindo ou escondendo elementos

Com a diretiva nativa @if, espera que seja passada à ela uma lógica que retorna um booleano.

01-Criando nova propriedade show para guardar o valor booleano:

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({ ... })
export class MyFirstPage extends Page {

    public show: boolean = true;

    constructor() {
        super();
    }

    public toggleMsg() {
        this.render(() => {
            this.show = !this.show;
        });
    }

}

02- Aplicando a diretiva em dois elementos diferentes do template. No primeiro foi passado o valor real, no segundo foi passado o mesmo valor porem negado:


<div class="my-first-page">
    <p @if="show">Texto 01 exibido :)</p>
    <p @if="!show">Texto 02 exibido :D</p>
    <button (click)="toggleMsg()">
        Alternar
    </button>
</div>

03- Resultado:

fiber_manual_record
http://localhost:8090/my-first

Texto 01 exibido :)

Exibindo lista de elementos

Com a diretiva nativa @for, é possível reaproveitar um mesmo elemento para exibir cada item de uma lista.

01-Criando nova propriedade menuItems para guardar uma lista de dados:

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({ ... })
export class MyFirstPage extends Page {

    public menuItems: string[] = [
        'Home',
        'About',
        'Business',
        'Contact'
    ];

    constructor() {
        super();
    }

}

02-Esta diretiva possui uma instrução específica para que funcione corretamente, @for="item of list", muito similar à um for utilizado em um código JavaScript ou TypeScript:


<div class="my-first-page">
    <ul>
        <li @for="item of menuItems">
            {{item}}
        </li>
    </ul>
</div>

03- Resultado:

http://localhost:8090/my-first
