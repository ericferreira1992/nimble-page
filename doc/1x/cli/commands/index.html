Comandos interativos

Assim como toda CLI, a Nimble CLI também possui seu alias para usa-lo globalmente, que é nb. Praticamente todos os comandos e funcionalidades estão no modo interátivo dela, basta escrever em seu terminal apenas nb, dar ENTER e as opções aparacerão, simples assim:

        
$ nb

Veja abaixo, de forma detalhada, todas as funcionalidades e comandos.

Generate

O Generate é uma opção que permite gerar scripts e códigos de forma simplificada para cada finalidade, seja ao criar uma nova página, um novo diálogo, uma diretiva ou até mesmo uma classe de serviço injetável.

Após executar nb em seu terminal, será exibida as opções. A opção Generate é a qual será abordada, como mostrado abaixo:

            
$ nb
? Select what do you want: (Use arrow keys)
❯ Generate 
  Run server
  Run build 
  Exit

Ao selecionar a opção Generate e apertar a tecla Enter, apareça as opções abaixo, estas serão abordadas nos tópicos abaixo.

            
$ nb
? Select what do you want: Generate
? Which what do you want to generate? (Use arrow keys)
  Page
  Dialog
  Directive
  Service
Page

A opção Generate > Page permite criar uma estrutura pronta para uma nova página, única coisa que deverá ser feito após isto, é definir uma rota no esquema de rotas e referencia-la.

                
$ nb
? Select what do you want: Generate
? Which what do you want to generate? (Use arrow keys)
❯ Page
  Dialog
  Directive
  Service

Ao selecionar a opção Page e confirmar, será solicitado que digite o caminho aonde a página será criada (veja que possui um exemplo):

                
$ nb
? Select what do you want: Generate
? Which what do you want to generate? Page
? What is path and name? (ex.: pages/home)

Após informar o nome da página, a CLI criará todos arquivos de template, script e estilo:

                
$ nb
? Select what do you want: Generate
? Which what do you want to generate? Page
? What is path and name? (ex.: pages/home) pages/example
CREATED: src/app/pages/example/example.page.html
CREATED: src/app/pages/example/example.page.scss
CREATED: src/app/pages/example/example.page.ts

Server

O Run server é a opção que permite você "rodar" o Nimble em um simples servidor em NodeJs durante o desenvolvimento.

            
$ nb
? Select what do you want: (Use arrow keys)
  Generate 
❯ Run server
  Run build 
  Exit

Após confirmar a opção, o servidor de desenvolvimento mostrará algumas informações, pedirá para esperar um pouco e pronto! É só acessar este endereço http://localhost:8090 em seu navegador e verá seu site ou aplicação funcionando.

            
$ nb
? Select what do you want: Run server
❯ Nimble project is running at http://0.0.0.0:8090/ (open in your browser)
❯ Environment from: src/environments/env.local

❯ Await, starting the server...

Build

O Run build é a opção que permite você "gerar" (compilar) seu projeto Nimble para poder publica-lo, pois o projeto Nimble não é nativamente compatível com todos os navegadores, portanto a CLI vai gerar um código mais leve e com uma série de otimizações como minificão e ofuscação de códigos/scripts, remoção de comentários, atualização dos cashs de toda a aplicação entre outros, tudo para que ela rode com a maior performance possível após a publicação.

            
$ nb
? Select what do you want: (Use arrow keys)
  Generate 
  Run server
❯ Run build
  Exit

Após confirmar a opção, a CLI solicitará que escolha uma ENV, dentre as disponíveis no projeto, para ser usada durante o build.

            
$ nb
? Select what do you want: Run build
? Select the environment to be used: (Use arrow keys)
❯ local
  dev
  prod

Com a ENV selecionada, o build será iniciado em seguida. O tempo de build geralmente não demora nem 1 minuto, e o tempo dele é relativo ao tamanho do projeto.

            
$ nb
? Select what do you want: Run build
? Select the environment to be used: local

❯ Environment from: src/environments/env.local
❯ Await, starting the server...

✔ Compiled successfully!

❯ Directory where the build is: your-project/build

Build terminado com sucesso, a pasta "build" terá sido criada na raiz do projeto.
