Serviços (services)

Serviço (service) é um termo muito utilizado na programação para definir que uma classe em TypeScript deva ser utilizada para execer alguma função específica, seja para conter métodos que auxiliam no desenvolvimento ou até mesmo métodos que irão gerenciar dados ou informações que seriam utilizadas, por exemplo, para preencher alguma informação na página, sejam estes dados presentes numa LocalStorage ou até mesmo em um API (na nuvem), as utilidades são variadas.

Se você não sabe como criar um Service para seu projeto, veja como clicando aqui.

A estrtura

A estrutura das classes de serviço é muito simples, a diferença é que existe um decorador chamado @Injectable que fica acima da declaração da classe que o torna uma classe injetável em qualquer outra classe de serviço ou até mesmo em diretivas, páginas, diálogos, entre outros.

Neste decorador, é enviado um objeto com uma propriedade booleana chamada single que, caso seja definida como:

❯ true a instância da classe será única sempre que injetada (pode ser útil para persistir informações na classe enquanto o site ou aplicação está aberto).

❯ false uma nova instância da classe sempre será criada quando injetada (utilizado para reciclar qualquer informação que tenha nas instâncias).

Veja o exemplo de uma classe de serviço:

            
import { Injectable } from '@nimble-ts/core/inject';

@Injectable({ single: true })
export class ArticleService {

    constructor() {
    }

}

Como utilizar

Por mais simples que seja utilizar uma classe de serviço, é importante saber algumas coisas para que funcione corremante.

Importar no projeto

Depois de ter a classe de serviço criada, a primeira coisa a se fazer é importa-la na raiz do projeto, na inicialização do Nimble, dentro de providers, veja um exemplo:

                
import { NimbleApp } from '@nimble-ts/core';
import { ROUTES } from './app/routes';
import { ArticlesService } from './app/services/articles.service';

NimbleApp.config({
    routes: ROUTES,
    directives: [],
    providers: [
        ArticlesService
    ]
}).start()
Injetando para utilizar

Com a classe de serviço importada no projeto, agora é possível utiliza-la em classes de páginas, diálogos, directivas e também em outros serviços. Independente de onde a classe for utilizada, o padrão sempre o mesmo, bastando injeta-la no construtor, conforme é mostrado no exemplo abaixo:

                
import { Page, PreparePage } from '@nimble-ts/core/page';
import { ArticlesService } from 'src/app/services/articles.service';

@PreparePage({
    template: require('./example.page.html'),
    style: require('./example.page.scss')
})
export class ExamplePage extends Page {

    constructor(
        private articlesService: ArticlesService
    ) {
        super();
    }

}
