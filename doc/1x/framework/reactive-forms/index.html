Formulário reativo

Trabalhar e controlar formulários e ainda mantendo segurança dos dados, é sempre uma tarefa difícil. Nimble desenvolveu uma estrutura para trabalhar com formulários de forma muito facilitada, principalmente para trabalhar com validações de formulário que cada caso exige, sem contar na sua reatividade que o torna mágico.

A estrutura

Antes de iniciar com os forumlários do Nimble, é importante entender que mesmo que você trabalhe em seu template utilizando elementos form, input, select e etc, é preciso montar uma estrutura para que estes elementos comuniquem com seu código (script), para que possa ser feito o famoso two way data binding e fazer com que você tenha mais controle sobre os dados que estão sendo inseridos no formulário.

Form

Para criar um formulário, é necessário primeiro criar uma instância da classe Form, veja a estrutura da classe abaixo para entende-la:

                
export class Form {

    // Observer para escutar quando o formulário for submetido
    public onSubmit: Observer<Event>

    // Os form-fields do formulário
    public get fields(): { [field: string]: FormField }

    // Indica se o formulário está blurred (entrou e saiu de algum campo)
    public get blurred(): boolean

    // Indica se o formulário está touched (algum campo foi tocado)
    public get touched(): boolean

    // Verifica se o formulário está todo válido
    public get isValid(): boolean

    // Retorna os erros que o formulário possui
    public get errors(): { [name: string]: any } | null;

    // Formulário em formato de objeto com todas as propriedades com seus valores
    public value: { [name: string]: any }

    // Retorna um form-field pelo nome
    public get(fieldName: string): FormField;

    // Verifica se existe um campo específico no formulário
    public has(fieldName: string): boolean

    // Verifica se possui erros
    public hasErrors(): boolean

    // Reseta os valores de todos os campos do formulário e também as propriedades touched e blurred
    public reset(options?: { noNotify?: boolean, noUpdateElement?: boolean }): void

    // Valida todo o formulário e retorna um boolean do resultado
    public validate(): boolean

    // Define o formulário inteiro como touched
    public setTouched(): void

    // Define o formulário inteiro como untouched
    public setUntouched(): void

    // Define o formulário inteiro como blurred
    public setBlurred(): void

    // Define o formulário inteiro como unblurred
    public setUnblurred(): void

}
FormField

Todo formulário possui campos, e para instânciar a classe Form é preciso informar estes campos, que possuem propriedades baseadas na classe FormFieldPrepare, veja sua estrutura abaixo:

                
export class FormFieldPrepare {

    // Valor inicial do campo
    public value: string | number | boolean | null | undefined;

    // Lista de validadores: (opcional)
    public validators?: ((formField: FormField) => any)[];

}

Começar a usar

Os formulários reativos são bem fáceis de se usar, mas para que funcione perfeitamente e possa extrair o máximo que ele oferece, deverá primeiro realizar os procedimentos mínimo, que são bem simples.

01 - Criando o formulário

A primeira coisa que deve ser feita, é cria a instância do formulário a partir da classe Form (a que foi vista no tópico anterior) juntamente com seus campos, e deixa-la salva numa propriedade da sua página ou diálogo.

No exemplo abaixo iremos criar um formulário de autenticação (login). Perceba que o formulário está sendo criado logo no construtor da página (mas nada impediria de cria-lo depois), e também está sendo criado um método chamado onSubmit:

                
@PreparePage({
    template: require('./login.page.html'),
    style: require('./login.page.scss')
})
export class LoginPage extends Page {

    public loginForm: Form;

    constructor() {
        super();
        this.loginForm = new Form({
            username: { value: '' },
            password: { value: '' },
        });
    }
}
02 - Preparando o formulário no template

Depois de criar o formulário na classe da página, precisará agora criar o HTML do formulário no arquivo de template. Neste momento, existem 2 pontos muito importantes que são:

❯ Inserir a diretiva form no elemento form, definindo como valor a instância do formulário criado.

❯ Inserir em cada campo (input) a diretiva form-field-name, definindo como valor o nome do respectivo campo (os mesmos definidos na criação do formulário).

Dando continuidade ao exemplo anterior, veja como ficaria o formulário no template:

                
<div class="login-page">
    <form [form]="loginForm">
        <div class="form-group">
            <label>Username</label>
            <input class="form-control" form-field-name="username" type="text" />
        </div>

        <div class="form-group">
            <label>Password</label>
            <input class="form-control" form-field-name="password" type="password" />
        </div>

        <button type="submit">
            ENTER
        </button>
    </form>
</div>
03 - Submetendo e resgatando conteúdo

Depois de tudo pronto, será necessário saber quando o formulário for submetido e também poder intercepta-lo. Para isso, será necessário criar um novo método na página e inserir a diretiva (submit) no elemento form contendo a chamada do método. Neste exemplo, o método chamará onSubmit(), veja como fica a classe e o template:

                
@PreparePage({
    template: require('./login.page.html'),
    style: require('./login.page.scss')
})
export class LoginPage extends Page {

    public loginForm: Form;

    constructor() {
        super();
        this.loginForm = new Form({
            username: { value: '' },
            password: { value: '' },
        });
    }

    public onSubmit() {
        let formValue = this.loginForm.value;
        console.log(formValue);
        // Console result: { username: '', password: '' }
    }
}
                
            
                
<div class="login-page" (submit)="onSubmit()">
    <form [form]="loginForm">
        <div class="form-group">
            <label>Username</label>
            <input class="form-control" form-field-name="username" type="text" />
        </div>

        <div class="form-group">
            <label>Password</label>
            <input class="form-control" form-field-name="password" type="password" />
        </div>

        <button type="submit">
            ENTER
        </button>
    </form>
</div>

Validar formulários

Assim como é fácil criar formulários, definir validações para os campos também é algo muito simples, acompanhe logo abaixo.

01 - Criando o formulário

Utilizando o mesmo exemplo dos tópicos acima, na criação do formulário vamos acrescentar a cada campo a propriedade validators, inserindo nela uma lista de validações (podendo ser apenas uma ou várias). Veja que neste caso foi usada a classe (nativa do Nimble) Validators para inserir o validador de obrigatoriedade, mas que nesta classe também possuem várias outras validações prontas quem podem ser utilizadas. Veja também como ficou o método onSubmit() agora.

                
import { Page, PreparePage, Form, Validators } from '@nimble-ts/core';

@PreparePage({
    template: require('./login.page.html'),
    style: require('./login.page.scss')
})
export class LoginPage extends Page {

    public loginForm: Form;

    constructor() {
        super();
        this.loginForm = new Form({
            username: { value: '', validators: [ Validators.required ] },
            password: { value: '', validators: [ Validators.required ] },
        });
    }

    public onSubmit() {
        if (this.loginForm.isValid) {
            alert('O formulário está válido!');
        }
    }
}
02 - Preparando o formulário no template

Um dos pontos positivos de se usar as validações é a facilidade de se exibir os erros, da forma que você quiser. Veja abaixo como é fácil exibir e customizar mensagens de validações através dos recursos do formulário e de template do Nimble:

                
<div class="login-page">
    <form [form]="loginForm" (submit)="onSubmit()">
        <div class="form-group">
            <label>Username</label>
            <input class="form-control" [class]="{ 'is-invalid': loginForm.get('username').hasErrors() }" form-field-name="username" type="text"/>
            <p @if="loginForm.get('username').hasError('required')" class="invalid-feedback">
                Campo obrigatório
            </p>
        </div>

        <div class="form-group">
            <label>Password</label>
            <input class="form-control" [class]="{ 'is-invalid': loginForm.get('password').hasErrors() }" form-field-name="password" type="password"/>
            <p @if="loginForm.get('password').hasError('required')" class="invalid-feedback">
                Campo obrigatório
            </p>
        </div>

        <button type="submit">
            ENTER
        </button>
    </form>
</div>
