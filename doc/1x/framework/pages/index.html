Páginas

As páginas são o coração de um projeto Nimble, as quais são vinculadas às rotas, que por sua vez se tornam orquestradores de todo o seu site ou aplicação web.

No Nimble, uma página pode representar (literalmente) uma página inteira, porem em casos mais comuns várias páginas podem, no final, compor o resultado de uma página inteira. Para entender melhor, visualize a imagem que mostra uma árvore de rotas que compoem páginasaqui.

Se você tem dúvidas de como criar páginas para seu projeto, clique aqui.

A estrutura

Toda página possui uma estrutura que se divide em três partes, elas são template, estilo e código, respectivamente arquivos .scss, .html e .ts, todos os arquivos que compoem a página possuem o sufixo page. Veja com detalhe as responsabilidades de cada um:

❯ example.page.html é responsável por conter a estrutura em HTML, que pode se comunicar diretamente com o script.

<div class="example-page">
    example works!
</div>

❯ example.page.scss é responsável por dar vida e cor a estrutura HTML.

.example-page {
}

❯ example.page.ts é aonde fica o script, responsável por toda a parte programável da página, também é onde se torna possível criar regras para controlar, em tempo real, a estrutura da página.

import { Page, PreparePage } from '@nimble-ts/core/page';

@PreparePage({
    template: require('./example.page.html'),
    style: require('./example.page.scss')
})
export class ExamplePage extends Page {

    constructor() {
        super();
    }

}

O ciclo de vida

O ciclo de vida das páginas são controlotadas pela árvore de páginas das rotas, onde uma página é construída e renderizada uma após a outra, porem todas possuem um mesmo ciclo de vida, os quais podem ser escutados através da implementação dos métodos onEnter, onInit, onDestroy e onExit. Veja o que significa cada método e quando são invocados:

❯ onEnter é invocado quando o a rota referente a página é encontrada e construída, mas ainda não foi renderizada.

❯ onInit é invocado logo após o onEnter, quando a página é renderizada na tela, isso significa que os elementos do template foram renderizados no DOM.

❯ onDestroy é invocado quando o elementos da página são removidos do DOM.

❯ onExit é invocado para indicar a saída da página, devido a sua rota não pertencer à rota atual.
